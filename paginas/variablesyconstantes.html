Nomenclatura de Variables y Constantes

¿Por qué es importante?

El uso adecuado de nombres para variables y constantes mejora la legibilidad del código y facilita su mantenimiento por parte de otros desarrolladores o del propio autor en el futuro.

Buenas prácticas

  • 🔸 Usar nombres descriptivos, como totalVentas o nombreUsuario.
  • 🔸 Emplear camelCase para variables en lenguajes como JavaScript, Java o C#.
  • 🔸 Para constantes, utilizar mayúsculas y guiones bajos: MAX_INTENTOS.
  • 🔸 Evitar abreviaciones excesivas o letras sueltas (ej: x, tmp).
  • 🔸 Alinear los nombres con el contexto del dominio del problema.

Ejemplos

  • let totalPagar = 0; ✅ Correcto
  • let tp = 0; ❌ Incorrecto (poco claro)
  • const LIMITE_TIEMPO = 30; ✅ Constante con estilo correcto
