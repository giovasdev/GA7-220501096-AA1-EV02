Nomenclatura de Variables y Constantes

Variables

Las variables almacenan datos que pueden cambiar durante la ejecución del programa. Una nomenclatura clara y coherente mejora la legibilidad, mantenimiento y comprensión del código.

  • ✏️ Utilizar camelCase: primera palabra en minúscula y siguientes en mayúscula. Ej: nombreUsuario, totalFactura.
  • ✏️ Usar nombres significativos que indiquen claramente el contenido o propósito. Ej: cantidadProductos, fechaNacimiento.
  • ✏️ Evitar abreviaciones confusas o letras sueltas como x, tmp (excepto en bucles o contextos muy breves).
  • ✏️ Prefiere sustantivos, ya que representan cosas o datos: precioUnitario, correoCliente.
  • ✏️ Mantén coherencia con el idioma del proyecto (español o inglés, pero no ambos).
// Correcto
let nombreCompleto = "Ana María";
let edadUsuario = 25;

// Incorrecto
let n = "Ana";
let e = 25;

Constantes

Las constantes almacenan valores que no deben cambiar durante la ejecución del programa. Usar una nomenclatura adecuada facilita su identificación inmediata.

  • 🔒 Usar mayúsculas separadas por guiones bajos: TASA_IVA, MAXIMO_INTENTOS.
  • 🔒 El nombre debe representar un valor fijo y ser claro en su propósito.
  • 🔒 Evitar usar constantes con nombres ambiguos o genéricos.
  • 🔒 Declarar constantes al inicio de archivos o funciones cuando sea posible.
// Correcto
const PI = 3.1416;
const TASA_IVA = 0.19;

// Incorrecto
const iva = 0.19;
const pi = 3.1416;
